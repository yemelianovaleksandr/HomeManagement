#  HomeManagement — Система управління будинком

Консольний застосунок для управління мешканцями та квартирами житлового будинку. Дозволяє вести облік мешканців, відслідковувати заселеність квартир та виконувати операції переселення з підтримкою транзакцій.

---

## Можливості

- **Мешканці** — додавання, перегляд та м'яке видалення (soft delete)
- **Квартири** — перегляд характеристик (тип, поверх, площа), деталі із списком мешканців
- **Операції** — закріплення мешканця за квартирою, переселення між квартирами (транзакційно)
- **Звіти** — список вільних квартир, ТОП-5 найзаселеніших квартир
- **Експорт** — вивантаження даних про мешканців у формат JSON та CSV

---

## Технологічний стек

- **Мова:** Python 3.10+
- **База даних:** PostgreSQL
- **Бібліотеки:** `psycopg2-binary`, `python-dotenv`

---

## Встановлення та запуск

### 1. Клонування репозиторію

```bash
git clone https://github.com/yemelianovaleksandr/HomeManagement.git
cd HomeManagement
```

### 2. Встановлення залежностей

```bash
pip install -r requirements.txt
```

### 3. Налаштування бази даних

Скопіюйте файл прикладу та заповніть своїми даними:

```bash
cp .env.example .env
```

Відредагуйте `.env`:

```env
DB_NAME=home_management
DB_USER=your_user
DB_PASSWORD=your_password
DB_HOST=localhost
DB_PORT=5432
```

> PostgreSQL має бути запущений. База даних створюється автоматично при першому запуску. Таблиці та демонстраційні дані також завантажуються автоматично.

### 4. Запуск

```bash
python main.py
```

```

├── main.py                   # Точка входу
├── requirements.txt
├── .env.example
├── sql/
│   ├── schema.sql            # Схема бази даних
│   └── seed.sql              # Демонстраційні дані
└── src/
    ├── config.py             # Конфігурація БД
    ├── database.py           # Підключення (Singleton)
    ├── models/               # Моделі даних
    ├── repositories/         # Робота з БД
    ├── services/             # Бізнес-логіка
    ├── ui/                   # Консольний інтерфейс
    └── utils/                # Валідація, логування, експорт
```

## Логи

Логи зберігаються у папці `logs/app.log` та ведуться автоматично під час роботи застосунку.

## Експорт даних

Файли експорту зберігаються у папці `exports/` з міткою часу у назві файлу.
